## What this article covers
This article explains the `flow-and-algorithm` view for `Panic, IO, and syscalls` in concrete, onboarding-first language.

## Syscall wrapper flow

1. Caller chooses typed selectors (`Descriptor`, `ReadLimit`) and creates `WriteRequest` or `ReadRequest`.
2. Corelib wrapper (`WriteWith` / `ReadWith`) resolves selectors via `match` into concrete fd and max-byte values.
3. Wrapper forwards to compatibility functions (`Write` / `Read`) that guard error conditions and call runtime builtins.
4. Runtime builtins (`__syscall_write`, `__syscall_read`) execute host interaction and return values mapped into `Core.Results.Result`.
