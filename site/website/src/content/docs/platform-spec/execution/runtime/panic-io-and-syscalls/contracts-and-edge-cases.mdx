## What this article covers
This article explains the `contracts-and-edge-cases` view for `Panic, IO, and syscalls` in concrete, onboarding-first language.

## Corelib syscall wrapper contracts

- `System.Syscall.Write(i64 fd, string data)` and `System.Syscall.Read(i64 fd, i64 maxBytes)` remain the compatibility layer over runtime builtins.
- New typed wrappers (`WriteWith`, `ReadWith`) must lower to those raw wrappers by resolving discriminated unions (`Descriptor`, `ReadLimit`) in corelib, not in user code.
- `Descriptor::Standard(StandardStream::Stdin|Stdout|Stderr)` must resolve to `0|1|2`; `Descriptor::Raw(fd)` must preserve the provided fd.
- `ReadLimit::Default` must resolve to a deterministic corelib default (`DefaultReadLimit()`), while `ReadLimit::UpTo(n)` must preserve `n`.
- Existing error guarantees remain unchanged: invalid fd yields `InvalidFd`, non-stdin read fd yields `UnsupportedReadFd`, and invalid read limit yields `InvalidReadLimit`.
