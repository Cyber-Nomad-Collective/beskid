## Model at a glance

`Corelib discovery and packaging` is modeled as a contract surface between producer crates and consumer crates. New contributors should treat it as a boundary document first and an implementation detail map second.

## Primary actors

- Producer side: crates that define or emit the contract surface.
- Consumer side: crates that read, execute, or validate that surface.
- Conformance side: test crates that pin expected behavior.

## Concrete crate anchors

- Canonical package root `compiler/corelib/beskid_corelib`
- Canonical package identity `corelib` (declared in `Project.proj`)
- Corelib path discovery in `compiler/crates/beskid_analysis/src/projects/graph/resolver.rs`
- CLI embedding/install support in `compiler/crates/beskid_cli/build.rs` and `compiler/crates/beskid_cli/src/corelib_runtime.rs`
- Integration checks in `compiler/crates/beskid_tests/src/projects/corelib`
