This article documents the **design model** for **Syntax domain model generation**.

## Language alignment
**Capture** predicates in `meta` compile to keys over syntax identities from this model. The host must regenerate or diff snapshots so **process**/**emit** never observe torn trees.

## Persistent entities
- **Compilation instance** — implicit handle to the compilation under construction (see language-meta meta block scope).
- **Syntax snapshot** — immutable tree with stable node identities suitable for incremental keys.
- **Capability tokens** — host-granted permissions for I/O, diagnostics, and emit operations inside meta.

## Boundaries
- Mod SDK facades never bypass the host bridge for effects.
- Generation logic in the reference compiler remains Rust-internal; Beskid sees only the generated `Beskid.Compiler.*` projection.

## Anchored code paths
- `compiler/crates/beskid_analysis/src/syntax/` — syntax item builders and node shapes.
- `compiler/crates/beskid_analysis/src/beskid.pest` — grammar surface feeding parse output.
- `compiler/crates/beskid_analysis/src/syntax/items/` — item-level parsers and metadata.

## Stable identities for capture and incremental keys

The syntax domain model **must** assign each node a **stable identity** within a `syntax_generation_id` window: identities **must** survive trivia edits when language law marks them stable, and **must** change when structural shape changes in ways that affect capture predicates. Identities **must** be comparable in O(1) time for incremental hashing.

**Torn tree prohibition** — Hosts **must not** expose partial syntax trees to `meta` process or emit: parsing failures for any required module in `entryModules` **must** abort meta registration for that round with diagnostics, leaving the previous committed generation untouched.

**Multi-root workspaces** — When multiple module roots exist (`CompilationContext::module_roots`), identities **must** be qualified by absolute normalized root path + in-root span so different projects cannot collide in capture caches.
