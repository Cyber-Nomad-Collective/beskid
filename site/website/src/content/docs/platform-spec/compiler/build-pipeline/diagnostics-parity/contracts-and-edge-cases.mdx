- Semantic error presence/absence must match across CLI and LSP for equivalent source.
- Source label differences are allowed when paths intentionally differ (`path`, `source.bd`, `"<memory>"`).
- Manifest (`.proj`) diagnostics must use project-error mapping paths, not code-file parsers.
- Snapshot reuse in LSP must not hide newly introduced semantic errors.
