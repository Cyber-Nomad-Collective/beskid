1. Resolve source path/text through services (manifest, workspace graph, `CompilePlan`, optional materialized workspace).
2. Parse program and produce syntax diagnostics.
3. **Meta host (when `Meta` projects attach to this compilation)** — Discover `meta` items in deterministic order, run capture → process → emit under **[meta block host bridge](/platform-spec/compiler/metaprogramming-mod-sdk/meta-block-host-bridge/)** policy, merge contributions, then re-parse as required by the active round (see design model for bounded iteration). Skip entirely when no attached `Meta` projects or host disables meta for this command.
4. Run builtin staged semantic rules when diagnostics are enabled (same diagnostic channel as meta diagnostics).
5. Abort on error diagnostics.
6. Transform to HIR, resolve symbols, and type-check.
7. Lower typed program to backend artifact for JIT/AOT.

The canonical lowering entrypoint remains `beskid_codegen::services::lower_source` (and `_with_pipeline`); meta orchestration **must** live in `beskid_analysis` services (or a dedicated host module re-exported there) so CLI, LSP, and codegen share one scheduling spine.
