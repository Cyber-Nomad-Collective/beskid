## Model at a glance

`Test harnesses and fixtures` is modeled as a contract surface between producer crates and consumer crates. New contributors should treat it as a boundary document first and an implementation detail map second.

## Primary actors

- Producer side: crates that define or emit the contract surface.
- Consumer side: crates that read, execute, or validate that surface.
- Conformance side: test crates that pin expected behavior.

## Concrete crate anchors

- `compiler/crates/beskid_tests/src/analysis` fixture-driven semantic assertions
- `compiler/crates/beskid_tests/src/runtime` runtime behavior checks
- `compiler/crates/beskid_e2e_tests/src/tests/runtime_cases.rs` source-to-runtime outcomes
- `compiler/crates/beskid_tests/src/doc_tests.rs` docs-driven verification
