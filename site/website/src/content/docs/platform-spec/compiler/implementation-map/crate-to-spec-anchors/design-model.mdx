## Model at a glance

`Crate-to-spec anchors` is modeled as a contract surface between producer crates and consumer crates. New contributors should treat it as a boundary document first and an implementation detail map second.

## Primary actors

- Producer side: crates that define or emit the contract surface.
- Consumer side: crates that read, execute, or validate that surface.
- Conformance side: test crates that pin expected behavior.

## Concrete crate anchors

- `beskid_analysis` -> parser/resolution/semantic leaves
- `beskid_codegen` -> lowering contract leaves
- `beskid_abi` and `beskid_runtime` -> execution ABI/runtime leaves
- `beskid_tests` and `beskid_e2e_tests` -> conformance leaves
