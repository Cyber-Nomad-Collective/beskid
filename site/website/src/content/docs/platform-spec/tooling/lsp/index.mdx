<SpecSection title="Meta / compiler mods" id="meta-lsp">
When the workspace includes **`Meta` projects**, the language server **must** use the same `CompilationContext` and meta invalidation rules as the CLI (**[Diagnostics and workspace analysis](/platform-spec/tooling/lsp/diagnostics-and-workspace-analysis/)**, **[Snapshot and refresh contract](/platform-spec/tooling/lsp/snapshot-and-refresh-contract/)**). Status phases **should** reuse the `beskid_pipeline` vocabulary surfaced through CLI observers so extension UX can show meta progress consistently.
</SpecSection>
