## What this article covers
This article explains the `flow-and-algorithm` view for `Project manifest contract` in concrete, onboarding-first language.

## Meta projects in the load flow
1. Discover `Project.proj` and parse manifest.
2. Classify **project type**; if **`Meta`**, validate required **meta** sections (attachment targets, entry modules—normative keys TBD in examples/contracts).
3. Include the project in the workspace graph so **[Workspace resolution contract](/platform-spec/compiler/resolution-and-projects/workspace-resolution-contract/)** exposes it to CLI and compiler services.
4. During compilation, the **meta host** consumes the classified **`Meta`** node list and registers **event subscriptions** per **[Metaprogramming Mod SDK](/platform-spec/compiler/metaprogramming-mod-sdk/)**.

## Newcomer walk-through
Start from the parent feature hub to learn the contract boundaries, then map the behavior to the listed implementation anchors before reading test coverage.
