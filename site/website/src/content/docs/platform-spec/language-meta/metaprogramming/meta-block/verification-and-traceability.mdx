This article documents **verification and traceability** for **`meta`** and **`Meta` projects**.

## Traceability

- **Grammar** — `compiler/crates/beskid_analysis/src/beskid.pest` and `syntax/items/` parsers for `meta` and project manifests.
- **Resolution** — `compiler/crates/beskid_analysis/src/resolve/` and workspace services for **`Meta`** project discovery.
- **Host / scheduling** — `compiler/crates/beskid_analysis/src/services.rs` and Mod SDK host features under **[Metaprogramming Mod SDK](/platform-spec/compiler/metaprogramming-mod-sdk/)**.
- **Diagnostics** — `compiler/crates/beskid_lsp/src/diagnostics.rs` and analysis diagnostic kinds for parity with meta-reported issues.

## Checklist

- [ ] `Meta` project type parsed and validated; stable diagnostics on invalid manifests.
- [ ] `meta` item supports capture/process/emit (or documented lowering) without ambiguity.
- [ ] Deterministic ordering tests shared with **[incremental scheduling](/platform-spec/compiler/metaprogramming-mod-sdk/incremental-scheduling-determinism/)** contract.
- [ ] Scope rules align with **[name resolution](/platform-spec/language-meta/program-structure/name-resolution/)**.
