<SpecSection title="Language-first model" id="language-first">
Compile-time metaprogramming is anchored on two language-level ideas:

1. **`Meta` project type** — A `Project.proj` whose **project type** is `Meta` participates directly in the compiler’s meta orchestration: the build loads that project so its `meta` declarations can subscribe to compilation events and drive declarative pipelines. Schema and wiring details are normative under **[Project manifest contract](/platform-spec/tooling/manifests-and-lockfiles/project-manifest-contract/)**; this page summarizes behavior users rely on.

2. **`meta` compilation block** — Each `meta` item is a **declaration of the process** the compiler must run: a **non-destructive, iterative** pipeline of **queries and typed transforms** over the **current compilation**, not an unconstrained imperative plugin. Normative grammar, phases, and guarantees live under **[meta compilation block](/platform-spec/language-meta/metaprogramming/meta-block/)**.
</SpecSection>

<SpecSection title="Declarative pipeline (capture, process, emit)" id="pipeline">
A `meta` block standardizes compiler interaction as three conceptual phases (exact surface syntax is specified in the meta-block feature):

- **Capture** — Declaratively bind **targets** (syntax nodes, symbols, or other stable handles) from the **entire compilation context** (sources, dependency graph, options), so the host knows what to watch for invalidation.
- **Process** — A **fluent pipeline** of queries and transformations over captured targets using `Beskid.Compiler.*` APIs (compiler contract). Outputs are **plans** (diagnostics, typed edits, contributions), not raw text patches.
- **Emit** — Apply contributions through the host so the merged program remains well-formed; diagnostics are reported through the **compiler-native** diagnostic channel so the Rust compiler and LSP can surface them consistently.

This shape supports **iterative** behavior: the host may run additional rounds until a fixed point or a diagnostic is raised, per language and host policy.
</SpecSection>

<SpecSection title="Hybrid canonical ownership" id="ownership">
- **[Language meta](/platform-spec/language-meta/)** — `Meta` projects, `meta` placement, scope, visibility, determinism, and the capture/process/emit user contract.
- **[Compiler / Metaprogramming Mod SDK](/platform-spec/compiler/metaprogramming-mod-sdk/)** — `Beskid.Compiler.Syntax`, `Query`, `Process`, `Emit`, `Diagnostics`, host bridge, merge semantics, and incremental scheduling.

Neither domain duplicates the other: language pages do not specify Rust algorithms; compiler pages do not redefine `Meta` or `meta` grammar.
</SpecSection>
