This **language-meta** feature hub defines the **`meta` compilation block**: a **declarative declaration of the process** the compiler runs—**capture** (which nodes and contexts are targeted), **process** (fluent queries and transforms), and **emit** (typed contributions and compiler-native diagnostics). It is the primary orchestration surface for **`Meta` projects** and for `meta` items loaded in other project kinds where the platform allows compile-time hooks.

Host execution, `Beskid.Compiler.*` facades, and incremental scheduling are under **[Metaprogramming Mod SDK](/platform-spec/compiler/metaprogramming-mod-sdk/)**. **`Meta` project type** manifest rules are under **[Project manifest contract](/platform-spec/tooling/manifests-and-lockfiles/project-manifest-contract/)**.

## Normative split
- **Here (language meta):** grammar, placement, lexical scope, visibility, capture/process/emit semantics, determinism, and non-destructive iteration.
- **Compiler domain:** Rust-side pipeline, query translation, diagnostic codes, and cache boundaries.
