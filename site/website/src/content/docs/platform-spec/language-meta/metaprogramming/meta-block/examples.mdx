This article provides **informative examples** (not normative syntax) for **`Meta` projects** and declarative **`meta`** pipelines.

## Meta project manifest (illustrative)

Illustrative `Project.proj` fragment:

```text
project:
  type: Meta
  # Host-defined: which compilation(s) this meta package attaches to
  meta:
    attachTo: [ "default" ]
```

Exact keys are defined under **[Project manifest contract](/platform-spec/tooling/manifests-and-lockfiles/project-manifest-contract/)**.

## Declarative capture → process → emit

Illustrative shape (keywords and braces may differ once grammar is frozen):

```beskid
meta MyProcess {
  capture {
    // Bind targets: e.g. all types implementing IFoo in current compilation
  }
  process {
    // Fluent Beskid.Compiler.Query pipeline; diagnostics here
  }
  emit {
    // Typed contributions only; host validates merge
  }
}
```

## Generator-style contributor

Capture binds interface declarations; process builds adapter bodies; emit adds methods to partial types allowed by the language.

## Analyzer-style inspector

Capture binds named patterns; process runs naming checks; emit is empty except for **diagnostics** on the compiler channel.

## Compilation inspection

Capture binds package references; process records structured facts via compiler APIs; no emit.
