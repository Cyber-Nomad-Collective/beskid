<SpecSection title="Scope" id="scope">
This area defines how Beskid programs interact with **foreign code** and with the **Rust-hosted runtime** that implements the language’s execution services.

- **[Interop.Contracts](/platform-spec/language-meta/interop/interop-contracts/)** — language-agnostic primitives (symbols, call shapes, ownership classes, conformance).
- **[C ABI profile](/platform-spec/language-meta/interop/c-abi-profile/)** — user **`Extern`** contracts targeting C-compatible libraries; Cranelift lowering and engine linking policy.
- **[Rust ABI profile](/platform-spec/language-meta/interop/rust-abi-profile/)** — stable runtime export surface and Rust-internal implementation boundaries.
- **[FFI and extern](/platform-spec/language-meta/interop/ffi-and-extern/)** — normative **language** chapter: where `Extern` may appear, what contracts mean to users, and how it ties to the profiles above.

Execution chapters under **`/platform-spec/execution/runtime/`** document lowering, syscalls, and operator policy; they **pair** with this area but do not replace it.
</SpecSection>
