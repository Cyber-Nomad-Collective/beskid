## Normative specification

This chapter defines naming and style constraints intended to be enforceable by formatter and lint tooling once promoted to Standard (normative) standing.

## Platform view

- Naming conventions shall be stable enough for generated docs and symbols.
- Formatting conventions shall align with `beskid_analysis` formatter output for code examples.
- Diagnostic messaging should use canonical term names from this chapter.
