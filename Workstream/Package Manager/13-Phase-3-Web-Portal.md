# Phase 3 — Web Portal (Blazor Server + Fluent UI Blazor)

## Outcomes
- Production-style UI flows for users.

## UI Areas
1. **Auth pages**
   - Register
   - Login
2. **Dashboard**
   - owned packages summary
   - recent publish history
3. **API Keys management**
   - create/revoke/list keys
4. **Package pages**
   - package detail
   - version list
   - publish guidance and API usage snippets

## Fluent UI Blazor Usage
- Use Fluent form controls, dialogs, tables, toasts
- Consistent design tokens and spacing scale
- Reusable components for forms and result states

## UX Criteria
- optimistic UI where safe
- clear error messages and validation
- mobile-friendly responsive layout

## Exit Criteria
- New user can complete all core tasks without raw API calls.
